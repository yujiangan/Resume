★ PersonaHub — AI 驱动的 GitHub 用户画像分析工具 AI
技术栈： TypeScript React 18 Nitro MiniMax ReAct Agent Tool Calling SSE

项目背景：

输入任意 GitHub 用户 ID，基于 LLM 与 ReAct 智能体自动分析其 timeline / repos / stars，几秒钟生成结构化技术画像报告，面向 HR 筛选、候选人评估、社区初识等场景

核心工作：

  • 独立完成 PRD、技术设计、前后端实现：撰写产品需求与技术设计文档（PRD / TECH-DESIGN），明确功能边界、异常路径与验收标准
  • 设计并实现 ReAct Agent 双阶段循环：工具阶段由 MiniMax 流式决策调用 GitHub 工具集，数据齐备后进入报告阶段按 Prompt 流式生成 Markdown 报告
  • 构建 Tool Calling 工程：定义工具 JSON Schema、抽象 dispatch 调度层，封装 get_profile / get_repos / get_events / get_stars 四个 GitHub 工具
  • 设计完整 SSE 事件协议（thinking_chunk / tool_start / tool_end / observation / report_chunk / report_done / error / done），前端 EventSource 命名事件分发，思考流与报告流互不打架
  • 基于 Nitro 3 搭建后端 API，封装 GitHubClient：自动分页（最多 1000 条事件）、AbortController 10s 单次超时、403 / 5xx 指数退避重试、X-RateLimit-Remaining 区分限流类型
  • 前端 React 18 + VitePlus，自定义 useAnalysis Hook 管理 SSE 连接，AgentStream 组件实时渲染思考过程与 Markdown 报告

技术亮点：

  • 深度实践 LLM 应用工程：双阶段 Agent 编排、Tool Calling 工程、结构化输出（PRD 四章对齐）、流式协议设计，具备完整的 AI Agent 产品落地能力
  • 系统级容错与降级：任一数据源失败不中断整次分析（profile / repos / events / stars 任一为空仍继续生成报告），全流程 60s 超时保护
  • 工程化细节扎实：单次请求 10s 超时 + AbortController、429/403/5xx 重试、字段筛选降低内存、流式不积累大数据
  • 前后端类型共享（src/shared/types.ts）+ 严格 TypeScript，跨端契约强一致
  • Vercel 一键部署（Nitro vercel preset），代码 90.1% TypeScript 占比

核心工作：
