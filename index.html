★ PersonaHub — AI 驱动的 GitHub 用户画像分析工具 AI
技术栈： TypeScript React 18 Nitro MiniMax ReAct Agent Tool Calling SSE

项目背景：

输入任意 GitHub 用户 ID（github.com/yujiangan/-PersonaHub），基于 LLM + ReAct 智能体拉取 profile / repos / events / stars，经多轮 Tool Calling + 流式报告生成，完整分析通常需数十秒；SSE 实时推送思考过程与 PRD 四章 Markdown 报告，面向 HR 筛选、候选人评估、社区初识等场景

核心工作：

  • 独立撰写 PRD / TECH-DESIGN 并完成前后端实现，明确功能边界、异常路径与验收标准
  • ReAct Agent 双阶段循环：MiniMax 流式选工具 → 数据齐备后按 REPORT_PROMPT 流式生成 Markdown 报告
  • Tool Calling 工程：JSON Schema 定义工具、dispatch 调度层，封装 get_profile / get_repos / get_events / get_stars
  • SSE 命名事件协议（thinking_chunk / tool_* / report_chunk / report_done / error / done），前端 EventSource 分发，思考区与报告区并行展示
  • GitHubClient 工程化：events·stars 最多 1000 条、repos 最多 500 条（遇空页即停）；单次 fetch 10s 超时；403/500/502/503 退避重试最多 2 次；X-RateLimit-Remaining 区分限流
  • React 18 + useAnalysis Hook + AgentStream：管理 SSE 连接，实时渲染思考流与 Markdown 报告

技术亮点：

  • LLM Agent 全链路落地：双阶段编排、Tool Calling、PRD 四章结构化输出、流式协议设计
  • 容错降级：单数据源失败不中断分析；全流程上限 60s（MAX_EXECUTION_MS），超时提示「分析超时，请重试」
  • 性能与资源控制：字段筛选减内存、SSE 流式不堆积、前后端共享 types（src/shared/types.ts）
  • Vercel 部署（Nitro vercel preset），仓库语言占比 90.1% TypeScript

★ 食刻 Timetaste — AI 美食推荐微信小程序 AI
技术栈： 微信小程序 TypeScript 通义千问 DashScope qwen-turbo qwen3-vl-plus z-image-turbo

项目背景：

「今天吃什么」决策小程序（github.com/yujiangan/timetaste），接入通义千问 DashScope，结合时间段 / 地理位置 / 采纳历史 / 上传菜单做个性化菜品推荐

核心工作：

  • AI 上下文工程（buildFoodRecommendContext）：整合早/午/茶/晚/夜宵时段、位置、最近 5 条采纳记录、SavedMenuList 动态拼装 Prompt
  • 三模型分工：qwen-turbo 文本推荐、qwen3-vl-plus 菜单/菜品识图、z-image-turbo 生成菜品配图（1024×1024）
  • 抽象 callAI 统一调用层：文本 / 视觉双模式，动态 temperature（识图 0.1 / 推荐 0.7），超时 15s / 30s / 60s
  • JSON 格式约束 + 容错解析：strip markdown 代码块 → JSON.parse → 正则提取兜底，保证小程序端稳定取数

技术亮点：

  • LLM 产品化实践：Prompt 工程、结构化 JSON 输出、多模态协同、分级超时与错误降级
  • 业务闭环：首页摇一摇推荐 → 菜单页识图入库 → 历史采纳反哺上下文，形成偏好闭环
  • utils 分层清晰（api.js / track.js），页面覆盖 home / menu / history / about

★ Imagedown — 图片批量下载工具
技术栈： Node.js TypeScript Fetch API Yargs GitHub Actions

项目背景：

Node.js 图片爬虫 CLI（github.com/yujiangan/imagedown），已发布 npm 包 @yujiangan/imagedown（v1.0.6），npx 一行命令从网页批量下载图片

核心工作：

  • TypeScript + Yargs 搭建 CLI：positional 参数（url / folder）、-c 并发数（默认 1）、help / strict 模式
  • Fetch API 抓取 HTML，正则提取 img src，按 Content-Type / URL 识别 JPG/PNG/GIF/WebP/SVG 等格式
  • 自研 concurrentTask 并发队列（worker + 任务池），控制下载并发、单任务失败不阻塞其余任务
  • 配置 GitHub Actions（.github/workflows），tag 触发自动构建与 npm 发布；维护 CHANGELOG，6 个 Release

技术亮点：

  • 并发下载：-c 参数线性提升吞吐，README 实测 Bing 图片场景
  • 内置 User-Agent / Referer 请求头，自动创建保存目录，提升站点兼容性
  • 完整 npm 包工程化：bin 入口、prepublishOnly 构建、files 白名单发布 dist
  • 仓库语言占比 98.4% TypeScript
