核心工作：

  • 独立完成 GitHub 用户画像分析工具从 PRD、技术设计、Agent 工具链到前端流式界面的落地，使用 Superpowers 拆解流式改造方案和任务计划，按小步骤约束 AI coding 并补充关键测试。

技术亮点：

  • 技术栈选择：前端采用 React + TypeScript 管理流式状态与工具卡片，服务端采用 Nitro API Routes 隔离 GitHub / LLM Token；长连接场景选择 SSE，减少轮询开销并提升实时反馈。

  • Agent 工程化：抽象工具调用协议与统一执行层，规范工具入参、执行结果、错误返回和过程日志，并通过迭代上限约束 Agent 行为，降低循环调用和不可控输出风险。
  • 质量保障：围绕流式解析、接口容错、SSE 通信和工具分发补充关键测试用例，保证核心链路在后续模型与工具迭代中具备可验证性和可维护性。

★ AI 美食推荐微信小程序 — 食选家（微信搜索食选家打开） AI
https://github.com/yujiangan/timetaste
技术栈： 微信小程序 TypeScript 千问大模型

项目背景：

食选家决策小程序，基于微信小程序原生能力接入 通义千问 DashScope，将文本推荐、菜单识别和菜品配图串成移动端闭环，结合时间段 / 地理位置 / 采纳历史 / 上传菜单做个性化菜品推荐。

核心工作：

  • 独立完成小程序推荐、菜单识别、历史记录与行为埋点等核心流程，覆盖从触发推荐、AI 返回、采纳 / 换菜到偏好沉淀的完整用户路径。

技术亮点：

  • 技术栈选择：使用 微信小程序 + TypeScript 贴合移动端入口和系统能力，按任务拆分 qwen-turbo 文本推荐、qwen3-vl-plus 菜单识别、z-image-turbo 菜品配图。
  • 体验优化：AI 请求与骰子动画、分阶段 loading 文案并行执行，展示时间分析 / 位置 / 偏好 / 菜单匹配过程，降低等待焦虑；同时设计错误卡片、重试推荐、权限兜底和结果卡片动效。
  • 个性化推荐闭环：记录用户的采纳 / 跳过行为、上传菜单和位置信息，换一换时避免重复推荐，形成“推荐 → 采纳 / 跳过 → 下次优化”的偏好闭环。
  • 稳定输出与性能兜底：针对文本推荐、菜单识别和图片生成等不同耗时链路设计分层超时、结构化解析兜底、图片缓存与压缩策略；图片生成失败时降级为默认占位展示，保证推荐文字主流程稳定可用。
  • 产品行为洞察：使用 wx.reportEvent 采集推荐、换菜、采纳、菜单上传 / 识别 / 保存等关键动作，为后续优化推荐效果和用户路径提供依据。
